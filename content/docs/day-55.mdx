# Day 55: Serverless Security

## 🎯 What You'll Learn

- Security risks in serverless
- Best practices
- Tools for securing serverless

## ⚠️ Security Risks

- Insecure function permissions
- Injection attacks
- Exposed environment variables

## ✅ Best Practices

- Principle of least privilege
- Validate all inputs
- Use environment variables securely

## 🛠️ Tools

- AWS IAM, Secrets Manager
- Third-party security scanners

## 📝 Hands-on Exercise

- Review permissions for a serverless function
- Add input validation to a function

## ❓ Assessment

- Name a serverless security risk
- How can you secure environment variables?

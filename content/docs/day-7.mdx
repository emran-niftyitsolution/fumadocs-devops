## ðŸ”¢ Advanced Subnetting

### Subnetting Fundamentals

**Binary to Decimal Conversion:**

```bash
# Understanding binary math
# 8-bit binary: 11111111 = 255
# 24-bit subnet mask: 11111111.11111111.11111111.00000000 = 255.255.255.0

# Quick conversion table:
# 128 64 32 16 8 4 2 1
# 1   1  1  1  1 1 1 1 = 255
# 1   1  0  0  0 0 0 0 = 192
# 1   1  1  0  0 0 0 0 = 224
```

**Subnet Mask Calculation:**

```bash
# /24 = 255.255.255.0 (256 addresses)
# /25 = 255.255.255.128 (128 addresses)
# /26 = 255.255.255.192 (64 addresses)
# /27 = 255.255.255.224 (32 addresses)
# /28 = 255.255.255.240 (16 addresses)
# /29 = 255.255.255.248 (8 addresses)
# /30 = 255.255.255.252 (4 addresses)
```

## ðŸ—ï¸ Network Design Principles

### Network Hierarchy

**Three-Tier Architecture:**

```bash
# Core Layer (Backbone)
# - High-speed switching
# - Redundancy and reliability
# - Minimal packet manipulation

# Distribution Layer (Aggregation)
# - Policy enforcement
# - Route summarization
# - VLAN routing

# Access Layer (Edge)
# - User connectivity
# - Port security
# - VLAN assignment
```

**Network Segmentation:**

```bash
# Separate networks by function:
# - Production: 10.0.1.0/24
# - Development: 10.0.2.0/24
# - Testing: 10.0.3.0/24
# - Management: 10.0.4.0/24
# - DMZ: 10.0.5.0/24
```

### VLAN Design

**VLAN Concepts:**

```bash
# VLAN (Virtual Local Area Network)
# - Logical network segmentation
# - Broadcast domain isolation
# - Security and performance benefits

# Common VLAN assignments:
VLAN 1:   Default/Native
VLAN 10:  Management
VLAN 20:  Production
VLAN 30:  Development
VLAN 40:  Testing
VLAN 50:  Guest
VLAN 100: Voice
VLAN 200: Video
```

**VLAN Configuration (Linux):**

```bash
# Create VLAN interface
sudo ip link add link eth0 name eth0.10 type vlan id 10

# Configure VLAN interface
sudo ip addr add 192.168.10.1/24 dev eth0.10
sudo ip link set eth0.10 up

# Remove VLAN interface
sudo ip link delete eth0.10
```

**VLAN Configuration (Network Switch):**

```bash

# Configure interface
interface FastEthernet0/1
 switchport mode access
 switchport access vlan 10

```

### Network Security Design

**Security Zones:**

```bash
# Internet (Untrusted)
# - External threats
# - No direct access to internal resources

# DMZ (Semi-trusted)
# - Public-facing services
# - Web servers, mail servers
# - Limited access to internal network

# Internal (Trusted)
# - User workstations
# - Internal applications
# - Controlled access

# Management (Highly trusted)
# - Network management
# - Administrative access
# - Restricted access
```

**Firewall Rules:**

```bash
# Basic firewall rules:
# Allow established connections
iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

# Allow SSH from management network
iptables -A INPUT -p tcp --dport 22 -s 10.0.4.0/24 -j ACCEPT

# Allow HTTP/HTTPS to DMZ
iptables -A INPUT -p tcp --dport 80,443 -d 10.0.5.0/24 -j ACCEPT

# Drop everything else
iptables -A INPUT -j DROP
```

## âš–ï¸ Load Balancing Concepts

### Load Balancer Types

**Application Load Balancer (Layer 7):**

```bash
# HTTP/HTTPS load balancing
# - Content-based routing
# - SSL termination
# - Health checks
# - Session persistence

# Example configuration:
upstream backend {
    server 192.168.1.10:8080 weight=3;
    server 192.168.1.11:8080 weight=3;
    server 192.168.1.12:8080 weight=3;
    server 192.168.1.13:8080 backup;
}
```

**Network Load Balancer (Layer 4):**

```bash
# TCP/UDP load balancing
# - Fast forwarding
# - No SSL termination
# - Connection-based routing
# - High performance

# Example configuration:
stream {
    upstream backend {
        server 192.168.1.10:3306;
        server 192.168.1.11:3306;
    }

    server {
        listen 3306;
        proxy_pass backend;
    }
}
```

### Load Balancing Algorithms

**Round Robin:**

```bash
# Distribute requests evenly
# Simple and effective
# No consideration for server capacity

upstream backend {
    server 192.168.1.10:8080;
    server 192.168.1.11:8080;
    server 192.168.1.12:8080;
}
```

**Weighted Round Robin:**

```bash
# Distribute based on server capacity
# Higher weight = more requests

upstream backend {
    server 192.168.1.10:8080 weight=3;
    server 192.168.1.11:8080 weight=2;
    server 192.168.1.12:8080 weight=1;
}
```

**Least Connections:**

```bash
# Send to server with fewest connections
# Good for long-lived connections

upstream backend {
    least_conn;
    server 192.168.1.10:8080;
    server 192.168.1.11:8080;
    server 192.168.1.12:8080;
}
```

**IP Hash:**

```bash
# Route based on client IP
# Ensures session persistence

upstream backend {
    ip_hash;
    server 192.168.1.10:8080;
    server 192.168.1.11:8080;
    server 192.168.1.12:8080;
}
```

### Network Automation

**Ansible Network Automation:**

```yaml
# network_config.yml
---
- name: Configure network devices
  hosts: network_devices
  gather_facts: no

  tasks:
    - name: Configure VLANs
      ios_vlans:
        config:
          - name: Production
            vlan_id: 20
          - name: Development
            vlan_id: 30
        state: merged

    - name: Configure interfaces
      ios_interfaces:
        config:
          - name: GigabitEthernet0/1
            description: Production VLAN
            enabled: true
        state: merged

    - name: Configure routing
      ios_static_routes:
        config:
          - address_families:

                routes:
                  - dest: 0.0.0.0/0
                    next_hops:
                      - forward_router_address: 192.168.1.1
        state: merged
```

**Python Network Automation:**

```python
#!/usr/bin/env python3
import paramiko
import time

def configure_switch(hostname, username, password, config_commands):
    """Configure network switch via SSH"""
    ssh = paramiko.SSHClient()
    ssh.set_missing_host_key_policy(paramiko.AutoAddPolicy())

    try:
        ssh.connect(hostname, username=username, password=password)
        channel = ssh.invoke_shell()

        for command in config_commands:
            channel.send(command + '\n')
            time.sleep(1)

        output = channel.recv(4096).decode()
        print(f"Configuration applied to {hostname}")
        print(output)

    except Exception as e:
        print(f"Error configuring {hostname}: {e}")
    finally:
        ssh.close()

# Example usage
config_commands = [
    'enable',
    'configure terminal',
    'vlan 20',
    'name Production',
    'exit',
    'interface vlan 20',
    'ip address 192.168.20.1 255.255.255.0',
    'no shutdown',
    'exit',
    'write memory'
]

configure_switch('192.168.1.1', 'admin', 'password', config_commands)
```

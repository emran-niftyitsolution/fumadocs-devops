# Day 84: GitOps

## 🎯 What You'll Learn

- GitOps principles
- ArgoCD implementation
- Declarative deployments
- Git as single source of truth

## 🔄 GitOps Principles

1. **Declarative:** Describe desired state
2. **Versioned:** Git as source of truth
3. **Automated:** Pull-based deployments
4. **Observable:** Monitor and alert

## 🚀 ArgoCD

GitOps continuous delivery for Kubernetes:

```yaml
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: guestbook
  namespace: argocd
spec:
  project: default
  source:
    repoURL: https://github.com/argoproj/argocd-example-apps
    targetRevision: HEAD
    path: guestbook
  destination:
    server: https://kubernetes.default.svc
    namespace: guestbook
  syncPolicy:
    automated:
      prune: true
      selfHeal: true
```

## 📋 Declarative vs Imperative

- **Declarative:** Describe what you want
- **Imperative:** Describe how to get there

## 🔍 Observability

- **Sync Status:** Are resources in sync?
- **Health Status:** Are resources healthy?
- **Drift Detection:** Manual changes detected

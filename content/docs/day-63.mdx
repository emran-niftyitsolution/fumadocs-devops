# Day 63: Kubernetes Monitoring

## 🎯 What You'll Learn

- Why monitor Kubernetes?
- Tools for monitoring (Prometheus, Grafana, etc.)
- Key metrics

## 📊 Why Monitor Kubernetes?

- Ensure cluster health
- Detect issues early
- Capacity planning

## 🛠️ Tools

- **Prometheus:** Metrics collection
- **Grafana:** Visualization
- **Kube-state-metrics:** Cluster state

## 📈 Key Metrics

- Pod health
- Node resource usage
- API server latency

## 📝 Hands-on Exercise

- Install Prometheus and Grafana in a cluster
- Create a dashboard for pod health

## ❓ Assessment

- Name a tool for Kubernetes monitoring
- Why is monitoring important?

# Day 64: Kubernetes Logging

## 🎯 What You'll Learn

- Why logging matters in Kubernetes
- Logging architecture
- Tools (ELK, Loki, Fluentd)

## 📝 Why Logging Matters

- Troubleshoot issues
- Audit activity
- Meet compliance

## 🏗️ Logging Architecture

- Application logs -> stdout/stderr
- Log collectors (Fluentd, Logstash)
- Centralized storage (Elasticsearch, Loki)

## 🛠️ Tools

- **ELK Stack:** Elasticsearch, Logstash, Kibana
- **Loki:** Grafana's log aggregation
- **Fluentd:** Log collector

## 📝 Hands-on Exercise

- Set up Fluentd and Loki in a cluster
- Query logs in Grafana

## ❓ Assessment

- Name a log collector for Kubernetes
- Why centralize logs?

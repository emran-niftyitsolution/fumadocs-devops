# üîÑ What is DevOps?

## Introduction

DevOps is a set of practices that combines software development (Dev) and IT operations (Ops) to shorten the development lifecycle while delivering features, fixes, and updates frequently in close alignment with business objectives.

## üèõÔ∏è History of DevOps

### The Traditional Waterfall Model

Before DevOps, software development followed the **Waterfall model**:

- Long development cycles (months/years)
- Separate teams for development and operations
- Manual deployment processes
- Limited communication between teams
- Slow feedback loops

### The Birth of DevOps

DevOps emerged in the late 2000s as a response to the limitations of traditional software development:

**2009**: Patrick Debois coined the term "DevOps" at the Agile 2009 conference
**2010**: The first DevOpsDays conference in Belgium
**2011**: "The Phoenix Project" novel popularized DevOps concepts
**2013**: "The DevOps Handbook" provided comprehensive guidance

## üéØ DevOps Principles

### 1. **Culture First**

- Collaboration between development and operations
- Shared responsibility for the entire application lifecycle
- Continuous learning and improvement
- Blameless post-mortems

### 2. **Automation**

- Automate everything possible
- Infrastructure as Code (IaC)
- Automated testing and deployment
- Continuous Integration/Continuous Deployment (CI/CD)

### 3. **Measurement**

- Monitor and measure everything
- Key performance indicators (KPIs)
- Feedback loops
- Data-driven decisions

### 4. **Sharing**

- Knowledge sharing across teams
- Open source contributions
- Documentation and best practices
- Cross-functional training

```
Plan ‚Üí Code ‚Üí Build ‚Üí Test ‚Üí Release ‚Üí Deploy ‚Üí Operate ‚Üí Monitor
```

## üîÄ Agile vs DevOps vs SRE

### **Agile**

- Focus: Software development methodology
- Goal: Rapid delivery of working software
- Practices: Scrum, Kanban, sprints
- Scope: Development team

### **DevOps**

- Focus: Development and operations collaboration
- Goal: Faster delivery with higher quality
- Practices: CI/CD, automation, monitoring
- Scope: Development + Operations teams

### **SRE (Site Reliability Engineering)**

- Focus: Reliability and performance at scale
- Goal: Balance reliability with feature velocity
- Practices: SLOs, error budgets, automation
- Scope: Operations + Engineering teams

## üèóÔ∏è DevOps Architecture

### **Three Ways of DevOps**

#### 1. **Flow (Systems Thinking)**

- Optimize the entire value stream
- Reduce handoffs and delays
- Visualize work in progress
- Limit work in progress

#### 2. **Feedback (Amplify Feedback Loops)**

- Shorten feedback cycles
- Create safety in the system
- Learn from failures
- Share knowledge

#### 3. **Continuous Learning and Experimentation**

- Create a culture of continuous learning
- Practice and master skills
- Experiment and take risks
- Learn from success and failure

## üìä DevOps Metrics

### **Deployment Frequency**

- How often code is deployed to production
- Target: Multiple times per day

### **Lead Time**

- Time from code commit to production deployment
- Target: Less than one hour

### **Mean Time to Recovery (MTTR)**

- Time to restore service after failure
- Target: Less than one hour

### **Change Failure Rate**

- Percentage of deployments causing failure
- Target: Less than 15%

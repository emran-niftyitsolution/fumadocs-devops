# 🐧 Linux Essentials for DevOps

## Introduction

Linux is the foundation of modern DevOps. Most cloud infrastructure, containers, and DevOps tools run on Linux systems. Understanding Linux fundamentals is essential for any DevOps engineer.

## 🏗️ Linux Architecture

### **Kernel**

- Core of the operating system
- Manages hardware resources
- Provides system calls to applications

### **Shell**

- Command-line interface
- Bash, Zsh, Fish, etc.
- Scripting and automation

### **File System**

- Hierarchical directory structure
- Everything is a file
- Permissions and ownership

## 📁 Filesystem Hierarchy

```
/                    # Root directory
├── bin/            # Essential user commands
├── boot/           # Boot loader files
├── dev/            # Device files
├── etc/            # Configuration files
├── home/           # User home directories
├── lib/            # Shared libraries
├── media/          # Removable media
├── mnt/            # Mount points
├── opt/            # Optional applications
├── proc/           # Process information
├── root/           # Root user home
├── run/            # Runtime data
├── sbin/           # System binaries
├── srv/            # Service data
├── sys/            # System information
├── tmp/            # Temporary files
├── usr/            # User programs
└── var/            # Variable data
```

## 🔐 File Permissions & Ownership

### **Permission Types**

- **Read (r)**: View file contents
- **Write (w)**: Modify file contents
- **Execute (x)**: Run file as program

### **Permission Groups**

- **Owner**: File creator
- **Group**: Group members
- **Others**: Everyone else

### **Permission Examples**

```bash
-rw-r--r-- 1 user group 1024 Jan 1 12:00 file.txt
# Owner: read, write
# Group: read
# Others: read
```

### **Common Permission Commands**

```bash
chmod 755 file      # rwxr-xr-x
chmod 644 file      # rw-r--r--
chmod +x script.sh  # Add execute permission
chown user:group file
```

## 🔌 SSH Basics

### **SSH Connection**

```bash
ssh user@hostname   # Connect to remote host
ssh -p 2222 user@hostname  # Custom port
ssh -i key.pem user@hostname  # Use key file
```

### **SSH Key Management**

```bash
ssh-keygen          # Generate key pair
ssh-copy-id user@hostname  # Copy public key
ssh-add key.pem     # Add key to agent
```

### **SSH Configuration**

```bash
# ~/.ssh/config
Host myserver
    HostName 192.168.1.100
    User ubuntu
    Port 22
    IdentityFile ~/.ssh/mykey.pem
```

## 📝 Shell Scripting Basics

### **Variables**

```bash
#!/bin/bash
NAME="DevOps Engineer"
echo "Hello, $NAME"
```

### **Conditionals**

```bash
if [ -f "file.txt" ]; then
    echo "File exists"
else
    echo "File not found"
fi
```

### **Loops**

```bash
# For loop
for i in {1..5}; do
    echo "Number: $i"
done

# While loop
while [ $count -lt 10 ]; do
    echo $count
    ((count++))
done
```

### **Functions**

```bash
function backup() {
    local source=$1
    local dest=$2
    cp -r "$source" "$dest"
    echo "Backup completed"
}
```

## 🛠️ DevOps-Specific Linux Skills

### **Service Management**

```bash
systemctl start service    # Start service
systemctl stop service     # Stop service
systemctl restart service  # Restart service
systemctl status service   # Check status
systemctl enable service   # Enable at boot
```

### **Log Management**

```bash
journalctl               # System logs
tail -f /var/log/syslog  # Follow log file
grep ERROR /var/log/app.log  # Search for errors
```

### **Network Configuration**

```bash
ip addr show             # Show IP addresses
ip route show            # Show routing table
nslookup domain.com      # DNS lookup
dig domain.com           # Detailed DNS info
```

### **Firewall Management**

```bash
ufw status              # Check firewall status
ufw allow 22            # Allow SSH
ufw allow 80            # Allow HTTP
ufw enable              # Enable firewall
```
